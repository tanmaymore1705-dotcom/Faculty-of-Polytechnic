>> 01 / 04 · Evolution

From slot auctions to a more flexible network

The original Polkadot design used parachain slot auctions — teams bid DOT to reserve a 2-year slot. Polkadot 2.0 is a set of upgrades that make this more flexible, affordable, and powerful.

BEFORE
  • Parachain slots via 2-year auctions
  • Teams lock millions of DOT
  • Only ~50 slots total
  • Rigid pricing
POLKADOT 2.0
  • Agile Coretime — buy compute time like cloud credits
  • More flexible pricing
  • Smaller teams can participate
RESULT
  • More parachains
  • More experimentation
  • The ecosystem opens up to solo developers and small teams
Already live: Coretime was activated via governance vote in 2024. New parachains now use Coretime by default. This was decided entirely by DOT holder vote — no company decision needed.

>> 02 / 04 · Agile Coretime

Compute time — like renting cloud credits

The relay chain has a fixed amount of "core" capacity — the number of parachain blocks it can process per relay chain block. Coretime is the name for buying access to that capacity.

BULK CORETIME

Buy a full month of continuous access.

Best for production parachains with steady traffic. Prices set by an on-chain sales algorithm based on demand.

ON-DEMAND CORETIME

Buy a single block's worth of core time.

Perfect for: testing, hackathon projects, low-traffic chains that only need to post occasionally.

The analogy: Polkadot is like AWS. Bulk Coretime = reserved instance (pay for a month, cheaper per unit). On-demand Coretime = pay-as-you-go instance (more expensive per block, but zero commitment).

>> 03 / 04 · What's coming next

JAM — Join-Accumulate Machine

JAM is Gavin Wood's proposal for Polkadot's next generation — a fundamental redesign of the relay chain. It is very early and very advanced. We introduce it for awareness, not understanding.

WHAT IS JAM?

A new protocol that generalises the relay chain into a programmable, distributed computer. Not just "connect chains" — run arbitrary computation across a decentralised network.

WHY IT MATTERS

JAM could make Polkadot as general-purpose as Ethereum while keeping the performance and security. Think: distributed supercomputer, not just multi-chain network.

WHERE IT IS NOW

Proposed in 2024 (the "Gray Paper"). Multiple teams implementing it. Not yet live. Will eventually replace the relay chain — but only after extensive testing and a governance vote.

Don't worry about understanding JAM today. It's cutting-edge research. What you DO understand — hashing, blocks, consensus, parachains — is the foundation JAM builds on.

>> 04 / 04 · Kusama

Kusama — Polkadot's wild cousin

WHAT IS KUSAMA?

A separate network running the same code as Polkadot, but with faster governance and lower security requirements. Teams test parachains on Kusama before deploying to Polkadot.

Token: KSM

WHY IT EXISTS

New features (like Coretime!) are tested on Kusama first. If they work well for months, they get proposed to Polkadot governance.

Kusama is the canary in the coal mine — it tests things so Polkadot doesn't have to.

Real money, real risk. KSM has real value. Kusama is not a testnet — it's a live network. But it moves faster and tolerates more breakage than Polkadot.
