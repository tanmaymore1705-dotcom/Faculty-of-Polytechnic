Part 1 — the block factory

blockchain.py — Part 1
import hashlib, time, random # ── HASHING ────────────────────────────────────────── def hash_block(block): content = (str(block["index"]) + block["timestamp"] + str(block["transactions"]) + block["previous_hash"] + str(block["nonce"])) return hashlib.sha256(content.encode()).hexdigest() # ── BLOCK CREATION ─────────────────────────────────── def make_block(index, transactions, previous_hash, nonce=0): block = { "index": index, "timestamp": time.strftime("%Y-%m-%d %H:%M:%S"), "transactions": transactions, "previous_hash": previous_hash, "nonce": nonce, "hash": "", } block["hash"] = hash_block(block) return block
NONCE INCLUDED We include nonce in the hash so mine_block() can iterate it.

Part 2 — mining and growing the chain

blockchain.py — Part 2
# ── MINING ─────────────────────────────────────────── def mine_block(index, transactions, previous_hash, difficulty=3): target = "0" * difficulty nonce = 0 while True: block = make_block(index, transactions, previous_hash, nonce) if block["hash"].startswith(target): return block nonce += 1 # ── CHAIN OPERATIONS ────────────────────────────────── def add_block(chain, transactions, difficulty=3): chain.append(mine_block(len(chain), transactions, chain[-1]["hash"], difficulty)) def is_valid_chain(chain): for i, b in enumerate(chain): if b["hash"] != hash_block(b): return False if i > 0 and b["previous_hash"] != chain[i-1]["hash"]: return False return True

>> Clearing the air

Five things people get wrong about blockchain

  1. "Blockchain is the same as Bitcoin" — Bitcoin is one blockchain. There are thousands. Polkadot is another.
  2. "Blockchain is unhackable" — The chain is tamper-evident, not unhackable. Smart contracts and wallets are frequently exploited.
  3. "All blockchains use PoW" — Polkadot, Ethereum 2.0, Cardano all use Proof of Stake variants.
  4. "Transactions are anonymous" — They're pseudonymous. All transactions are public; the address isn't linked to a name by default.
  5. "Blockchain solves everything" — It solves the trust/immutability problem. For most databases a regular SQL db is faster and cheaper.
