>> 01 / 04 · Wallets

A wallet doesn't hold coins — it holds keys

Your bank balance is a number in the bank's database. Crypto is different: coins exist as entries on the blockchain. Your "wallet" is really just a program that stores your private key and shows you your balance by reading the blockchain.

Analogy: The blockchain is a safety deposit vault at a public bank. Anyone can see the boxes. Only the person with the private key can open their box.

HOT WALLET

Connected to the internet. Convenient for frequent use. MetaMask, Polkadot.js extension, Trust Wallet.

COLD WALLET

Offline device (Ledger, Trezor). Safer for large amounts. Private key never touches the internet.

>> 02 / 04 · Key pairs

The asymmetric key pair

Every wallet has exactly two keys. They are mathematically linked — but knowing the public key tells you nothing about the private key.

PUBLIC KEY / ADDRESS

Your address on the blockchain. Share it freely — this is how people send you tokens. Like your bank account number.

>> 03 / 04 · Transactions

A transaction is a signed message

"Alice sends 10 DOT to Bob." But how does the network know it's really Alice? She signs the transaction with her private key. Anyone can verify the signature using her public key.

transaction.py
# A transaction in our simplified model transaction = { "from": "Alice_address", "to": "Bob_address", "amount": 10, "signature": "signed_with_alice_private_key", } # In our toy blockchain, we skip real cryptography. # We just trust the 'from' field. # Real blockchains verify the signature before accepting.
We simplify here. Real signature verification requires ECDSA (elliptic curve cryptography). We'll represent transactions as dicts with a "signature" field but not implement the actual crypto — that's a university-level topic.

def make_block(index, transactions, previous_hash): block = { "index": index, "timestamp": time.strftime("%Y-%m-%d %H:%M:%S"), "transactions": transactions, # list of dicts "previous_hash": previous_hash, "hash": "", } block["hash"] = hash_block(block) return block # Create two transactions txns = [ {"from": "Alice", "to": "Bob", "amount": 10}, {"from": "Carol", "to": "Dave", "amount": 5}, ] block = make_block(1, txns, "0" * 64) print(f"Block 1 contains {len(block['transactions'])} transactions")

>> Real world

In Polkadot, transactions are called extrinsics

WHAT WE CALL IT

transaction = {from, to, amount}

Simple movement of value between addresses.

POLKADOT'S TERM

"Extrinsic" = any external message sent to the chain. Transfer DOT, vote in governance, stake tokens, call a smart contract — all are extrinsics.

Go to Subscan → click any block → look at the "Extrinsics" tab. You'll see a list of operations exactly like our transaction dicts. Each one has a sender, an action, and arguments.
