>> 01 / 04 · The problem

In a decentralised network — who adds the next block?

Nobody owns the blockchain. There's no boss. Thousands of computers all want to add the next block. If it's free to add a block, anyone can spam the chain with fake transactions.

The solution: Make adding a block cost something — specifically, computation time. Make the computer do lots of useless work (find a special number) before it earns the right to add a block. This is Proof of Work.

>> 02 / 04 · The key idea

Mining = finding the right nonce

A nonce is just a number we add to the block. We keep changing it until the block's hash starts with enough zeros — that's the "difficulty target."

mining_concept.py
import hashlib def hash_with_nonce(data, nonce): content = data + str(nonce) return hashlib.sha256(content.encode()).hexdigest() # Try different nonces: print(hash_with_nonce("block data", 0)) # → 5d7b9a... (doesn't start with 0) print(hash_with_nonce("block data", 1)) # → a3c21f... (doesn't start with 0) print(hash_with_nonce("block data", 42)) # → 0c74bd... (starts with 0 — with difficulty=1 this is valid!)
The difficulty is the number of leading zeros required. Difficulty 1 = one zero. Difficulty 4 = four zeros (0000...). More zeros = exponentially harder.

>> 03 / 04 · The code

The miner — a loop that keeps guessing

miner.py
def mine_block(index, data, previous_hash, difficulty=3): target = "0" * difficulty # e.g. "000" for difficulty 3 nonce = 0 while True: block = { "index": index, "timestamp": time.strftime("%Y-%m-%d %H:%M:%S"), "data": data, "previous_hash": previous_hash, "nonce": nonce, "hash": "", } block["hash"] = hash_block(block) if block["hash"].startswith(target): print(f"Mined! nonce={nonce}, hash={block['hash'][:16]}...") return block nonce += 1 # Mine a block: b = mine_block(1, "Alice → Bob: 10 DOT", "0" * 64, difficulty=3)
NONCE FIELD  We add a nonce field to our block dict so it gets included in the hash calculation. Changing the nonce changes the hash.
DAY 26 / 40
>> 04 / 04 · Difficulty matters

More zeros = exponentially more guesses

DIFFICULTY 1
Target: "0..."

Takes 1–16 guesses on average. Instant.

DIFFICULTY 4
Target: "0000..."

Takes ~65,536 guesses on average. A few seconds.

DIFFICULTY 6
Target: "000000..."

Takes ~16 million guesses. Minutes on a laptop.

Bitcoin uses difficulty 22+ (22 leading hex zeros). The whole network burns gigawatts of electricity each second to solve this puzzle. Polkadot avoids this with Proof of Stake — we'll see that tomorrow.

>> The tamper cost

Now tampering is expensive, not just detectable

If you tamper with block 5 in a 1000-block chain, you have to:
  1. Recalculate block 5's hash to meet the difficulty target — costs lots of work.
  2. Recalculate block 6, 7, 8... all the way to block 1000 — each costs work.
  3. Do all this faster than the whole honest network is adding new blocks.
With 51%+ of the network's computing power, you could theoretically do it — this is the famous 51% attack. Bitcoin's enormous mining network makes this practically impossible.
