DAY 25 / 40
>> 01 / 03 · The rules

A chain is valid if two conditions hold

CHECK 1: HASH INTEGRITY

Every block's stored hash must equal what we get when we recalculate it right now. If even one character of the block was changed after it was created, the hash won't match.

CHECK 2: CHAIN LINK INTEGRITY

Every block's previous_hash must equal the actual hash of the block that came before it. This proves the blocks are in the correct order and none have been inserted or swapped.

Both checks must pass for every block — except the genesis block, which has no predecessor to check against.
DAY 25 / 40
>> 02 / 03 · The function

is_valid_chain() — the full validator

validate.py
def is_valid_chain(chain): for i in range(len(chain)): block = chain[i] # Check 1: stored hash must match recalculated hash if block["hash"] != hash_block(block): print(f"Block {i}: hash is invalid!") return False # Check 2: previous_hash must match prior block's hash if i > 0: # skip genesis block if block["previous_hash"] != chain[i - 1]["hash"]: print(f"Block {i}: link is broken!") return False return True # all checks passed chain = build_chain() print(is_valid_chain(chain)) # True
SKIP GENESIS  Block 0 has no predecessor, so we only run Check 2 starting from block 1 (i > 0).

>> 03 / 03 · The tamper test

Prove it catches cheating

tamper_demo.py
chain = build_chain() print("Before tamper:", is_valid_chain(chain)) # True # Secretly change block 1's data chain[1]["data"] = "Alice sends 9999 DOT to Bob" print("After tamper:", is_valid_chain(chain)) # Block 1: hash is invalid! # False # What about recalculating the hash too? chain[1]["hash"] = hash_block(chain[1]) # update block 1's hash print("After hash fix:", is_valid_chain(chain)) # Block 2: link is broken! # False — block 2 still stores the OLD hash of block 1
You can't fix it without fixing every block after it. And in a real blockchain with Proof of Work, recalculating every subsequent block would require enormous computation.

>> Real world anchor

What Polkadot validators actually check

OUR CHAIN

We check: (1) hash integrity, (2) chain link. Runs in milliseconds on a laptop. Great for understanding the concept.

POLKADOT VALIDATORS

Also check: digital signatures on every extrinsic (transaction), account balances, runtime logic, state transitions. The concept is the same; the implementation is vastly more complex.

The key insight: Polkadot validators are doing is_valid_chain() — just with hundreds of additional rules layered on top. You now understand the foundation.
