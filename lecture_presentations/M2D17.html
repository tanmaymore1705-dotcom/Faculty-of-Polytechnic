>> 01 / 05 · The mental model

Every character has a number

Think of a string as boxes in a row. Counting starts at 0, not 1. You can also count from the right with negative numbers.

Blue = index from the front (starts at 0). Purple = index from the back (starts at -1). "Python"[0] is 'P'; "Python"[-1] is 'n'.
>> 02 / 05 · Indexing

Grab one character with s[i]

indexing.py
word = "Python"

word[0]    # 'P'  — first character
word[2]    # 't'
word[-1]   # 'n'  — last character
word[-2]   # 'o'  — second from the end

word[10]   # IndexError — there's no box 10
The handy trick.
s[-1] is the cleanest way to get the last character — no need to know the length.
>> 03 / 05 · Slicing

Grab a range with s[start:stop]

Takes from start up to but not including stop. That "not including" trips up everyone at first.

slicing.py
word = "Python"   #  P y t h o n
                  #  0 1 2 3 4 5
word[0:2]   # 'Py'   — boxes 0 and 1 (NOT 2)
word[2:5]   # 'tho'  — boxes 2,3,4
word[:3]    # 'Pyt'  — no start = from 0
word[3:]    # 'hon'  — no stop  = to the end
word[:]     # 'Python' — a full copy
Quick length check: s[2:5] gives 5 - 2 = 3 characters. The math always works out to stop - start.
>> 04 / 05 · The third number

The step — s[start:stop:step]

step.py
s = "ABCDEFGH"

s[::2]    # 'ACEG' — every 2nd character
s[1::2]   # 'BDFH' — every 2nd, from index 1

s[::-1]   # 'HGFEDCBA' — REVERSED 🔁
THE FAMOUS ONE

s[::-1] is the Python idiom for reverse a string. A negative step walks backwards. You'll use it in the palindrome check.

>> 05 / 05 · Classic #1

Palindrome check

A palindrome reads the same forwards and backwards — level, madam, racecar. So: is the string equal to its own reverse?

palindrome.py
def is_palindrome(text):
    clean = text.lower().replace(" ", "")  # reuse D16!
    return clean == clean[::-1]            # reverse & compare

print(is_palindrome("Racecar"))     # True
print(is_palindrome("hello"))       # False
print(is_palindrome("Never odd or even"))  # True
NOTICE Yesterday's .lower() and .replace() team up with today's [::-1]. The skills stack.
>> Classic #2

Initials extractor

Take a full name, grab the first letter of each word, and stitch them into uppercase initials. Split (D16) + indexing (D17) together.

initials.py
def initials(full_name):
    letters = []
    for word in full_name.split():   # ['Aisha', 'Noor', 'Khan']
        letters.append(word[0])      # first char of each
    return "".join(letters).upper()  # 'ANK'

print(initials("aisha noor khan"))  # 'ANK'
Three skills, one function: .split() to break it up, word[0] to index, .join() to rebuild. This is what "Python in practice" looks like.
>> Common failures

Four mistakes you'll make today

  1. Starting from 1 — Python counts from 0, so the first char is s[0].
  2. Expecting s[2:5] to include index 5 — stop is excluded.
  3. s[10] on a short string → IndexError. (But slicing past the end is safe!)
  4. Trying s[0] = "X" to change a letter → TypeError; strings are immutable (build a new one).
