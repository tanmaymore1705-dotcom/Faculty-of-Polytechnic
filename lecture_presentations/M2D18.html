>> 01 / 05 · The key difference

Lists are mutable — they change in place

This is the opposite of strings. With a string you had to catch a new value. With a list, the methods change the same list directly.

strings_vs_lists.py
# String — must reassign:
name = name.upper()

# List — changes itself, NO reassign:
marks = [70, 85]
marks.append(90)     # marks is now [70, 85, 90]
# NOT: marks = marks.append(90)  ← returns None!
The trap.
marks = marks.append(90) sets marks to None and loses your list. Just call marks.append(90) on its own line.
>> 02 / 05 · Growing a list

Add items: .append() & .insert()

growing.py
queue = ["Aisha", "Rohit"]

queue.append("Sneha")        # ['Aisha', 'Rohit', 'Sneha'] — adds at END
queue.insert(0, "Vikram")    # ['Vikram', 'Aisha', 'Rohit', 'Sneha']
                            # insert(position, item) — squeezes in
99% of the time you want .append(). It's the workhorse: start with an empty list [] and append as you go — exactly what you did in the initials loop yesterday.
>> 03 / 05 · Shrinking a list

Remove items: .remove() & .pop()

shrinking.py
names = ["Aisha", "Rohit", "Sneha"]

names.remove("Rohit")  # remove by VALUE → ['Aisha','Sneha']

last = names.pop()    # remove by POSITION (default: last)
                      # last == 'Sneha', names == ['Aisha']
first = names.pop(0)  # pop a specific index, returns it
remove vs pop.
.remove("x") = "delete this value". .pop(i) = "delete the item at this position and hand it back to me".
>> 04 / 05 · Searching a list

Is it there? Where? How many?

searching.py
roll = ["Aisha", "Rohit", "Sneha", "Rohit"]

"Sneha" in roll        # True  — the membership test
"Karan" in roll        # False

roll.index("Rohit")      # 1 — position of FIRST match
roll.count("Rohit")      # 2 — how many times it appears
len(roll)               # 4 — how many items total
Always check before you act. if "Karan" in roll: protects you — calling .remove() or .index() on a missing value raises a ValueError.
>> 05 / 05 · Ordering a list

.sort() vs sorted() — know the difference

in_place.py
# .sort() — changes the list itself
marks = [85, 70, 92]
marks.sort()           # marks is now [70, 85, 92]
marks.sort(reverse=True) # [92, 85, 70]
new_copy.py
# sorted() — returns a NEW sorted list
marks = [85, 70, 92]
ranked = sorted(marks)  # [70, 85, 92]
# marks is UNCHANGED: [85, 70, 92]
Rule: list.sort() rearranges in place and returns None. sorted(list) leaves the original alone and gives you a fresh, sorted copy. Strings and numbers both sort naturally.
>> Tying it together

Walk the list, do something to each

Everything from Module 1 still works: loop over a list, and combine these new verbs inside the loop.

together.py
roll = ["aisha", "ROHIT", "sneha"]

# Clean every name (D16) and collect into a new list
clean = []
for name in roll:
    clean.append(name.title())   # build up the result

clean.sort()                     # alphabetical
print(clean, "·", len(clean), "students")
# → ['Aisha', 'Rohit', 'Sneha'] · 3 students
PATTERN Empty list → loop → append → sort. You'll use this shape constantly.
>> Common failures

Four mistakes you'll make today

  1. Writing marks = marks.append(x) — .append returns None, so you lose the list.
  2. .remove("x") when "x" isn't there → ValueError. Check with in first.
  3. Expecting sorted(marks) to change marks — it doesn't; catch the return.
  4. Confusing .remove(value) with .pop(index) — one takes a value, the other a position.
