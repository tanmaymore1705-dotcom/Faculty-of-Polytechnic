>> Part A · Concepts (pen and paper)

Sample questions — the format you'll see

  • In one sentence: why does changing a block's data break the chain?
  • What are the TWO checks that is_valid_chain() performs?
  • What is a nonce? What do we do with it in Proof of Work?
  • Name ONE difference between Proof of Work and Proof of Stake.
  • In Polkadot, what is the difference between a validator and a nominator?
These are representative — not the actual test questions. If you can answer all five above confidently, you are ready.
>> Part B · Coding (Codespaces)

Sample coding problems — the format you'll see

PROBLEM 1 — 15 MIN

Given the hash_block() and make_block() functions (provided in the starter file), build a chain of 3 blocks. The first block's data should be 'Genesis'. The second should be 'Alice sends 10 DOT to Bob'. The third: 'Bob sends 4 DOT to Carol'. Print the hash of each block.

PROBLEM 2 — 15 MIN

The chain below has been tampered with. Write is_valid_chain() and use it to find which block is broken. Print: the index of the broken block, what check failed (hash integrity or chain link), and the two hash values that don't match.

Starter file provided. You will have hash_block() and make_block() already written in the starter file — you only need to use them, not rewrite them from scratch.
>> Part C · Reflection (written)

One open-ended question — 10 minutes

Reflection Question

Open Subscan (polkadot.subscan.io) and find any block from the last hour. Choose one specific field from that block — for example, the Parent Hash, or one Extrinsic — and explain how it relates to something you built in your Python blockchain. Be specific: name the field, name the Python function or code line that does the same thing, and explain why they match.

There is no single correct answer. We are looking for evidence that you can connect what you built in Python to the real production chain.
>> Exam strategy

60 minutes — use them well

0:00 – 0:20
Part A first
Five concept questions. ~4 minutes each. Write in full sentences — partial credit for partial understanding.
0:20 – 0:50
Part B coding
15 minutes per problem. Read the problem fully before writing any code. Start with the function signature.
0:50 – 1:00
Part C reflection
Open Subscan. Pick a block. Write your answer. Even a half-page answer is fine if it shows understanding.
If stuck on Part B: write comments explaining what your code WOULD do, even if you can't get the syntax right. Comments showing correct reasoning earn partial credit.
